# ZMK Firmware для Jorne (nice!nano v2)

Конфигурация ZMK для беспроводной сплит-клавиатуры Jorne-WL v3.0.1.

- **ZMK**: v0.3.0
- **Board**: nice_nano_v2
- **Shields**: jorne_left, jorne_right
- **ZMK Studio**: включен (USB)

## Сборка

Push в GitHub запускает автоматическую сборку. Скачать прошивку: вкладка **Actions** -> артефакт `firmware`.

## Прошивка

Для каждой половины (сначала правая, потом левая):

1. Подключить по USB
2. Двойной клик RESET (или замкнуть RST + GND) — появится USB-диск
3. Скопировать `.uf2` файл на диск:
   - Правая: `jorne_right-nice_nano_v2-zmk.uf2`
   - Левая: `jorne_left-nice_nano_v2-zmk.uf2`
4. После обеих — однократно нажать Reset на каждой половине

**Сброс Bluetooth** (если не подключается): прошить `settings_reset-nice_nano_v2-zmk.uf2` на обе половины, затем основную прошивку. Удалить старые устройства в настройках BT.

## Концепция раскладки

Раскладка спроектирована для **двуязычной работы** (русский + английский) и **программирования** на 42 клавишах.

### Проблема русского языка на 42 клавишах

В русском алфавите 33 буквы, а стандартный QWERTY имеет только 26 буквенных позиций. При переключении ОС на русский (ЙЦУКЕН), «лишние» русские буквы занимают позиции знаков препинания:

| Клавиша | Английский | Русский |
|---------|-----------|---------|
| `;`     | точка с запятой | **Ж** |
| `'`     | апостроф | **Э** |
| `[`     | скобка | **Х** |
| `]`     | скобка | **Ъ** |

Буквы **Ж**, **Э** и **Х** — частые, поэтому клавиши `;` `'` `[` обязаны быть на основном слое. **Ъ** и **Ё** используются редко — они вынесены на слой символов (через `]` и `` ` ``).

### Решение: hold-tap на угловых клавишах

На 42 клавишах не хватает места для всех модификаторов и русских букв одновременно. Вместо home-row mods (которые вызывают случайные срабатывания) используются **hold-tap на угловых клавишах** — их нажимают только намеренно:

| Позиция | Тап (короткое нажатие) | Удержание |
|---------|----------------------|-----------|
| Левый outer top (слева от Q) | TAB | LGUI (Cmd/Win) |
| Левый home-row edge | ESC | LCTRL |
| Правый средний thumb | F24 (переключение языка через скрипт) | слой NAV |

Левый средний thumb — чистый `&mo 1` (слой SYM на удержании, без tap-действия).

BSPC и LALT — отдельные самостоятельные клавиши (без hold-tap): BSPC на правом крайнем thumb (безопасно зажимается для повтора), LALT на правом нижнем углу pinky.

Это не home-row mods — клавиши с hold-tap находятся на позициях, куда палец тянется целенаправленно. Ложных срабатываний нет.

Переключение языка перенесено с угловой клавиши на layer-tap правого среднего thumb'а: тап = F24, hold = слой NAV. BSPC переехал под правый большой палец. TAB живёт только в левом outer top (pos 1) — sym-thumb стал чистым `&mo 1` без tap-действия.

### Архитектура слоёв

| Слой | Назначение | Активация |
|------|-----------|-----------|
| L0 (base) | QWERTY / ЙЦУКЕН | по умолчанию |
| L1 (sym) | Символы + numpad-калькулятор | `lt 1 TAB` — левый средний thumb (hold) |
| L2 (nav) | Vim-навигация (одной рукой) | `lt 2 F24` — правый средний thumb (hold) |
| L3 (sys) | Медиа, BT, F-keys, система | оба layer-ключа одновременно |
| L4 (game) | Игровой слой | combo: LCTRL + BSPC (крайние thumb) |

L3 реализован через **conditional layer** — ZMK автоматически активирует его, когда зажаты оба layer-ключа.

L4 активируется **combo** из двух крайних thumb-клавиш: `LCTRL` (левый крайний) + `BSPC` (правый крайний) — одновременное нажатие. Тот же combo выключает игровой слой обратно.

╰──────────┴─────┴─┬───┴─┬───┴───┬─┴─────╯   ╰─────┴┬────┴┬────┴──┬──┴───┬─┴──────────╯
                   │LCTRL│  SYM  │SPACE│            │ENTER│🌐/NAV │ BSPC │

> На русской раскладке базовые `,` и `.` превращаются в `б`/`ю`. Чтобы получить настоящие `,` и `.` на русском **на тех же физических позициях что и в английском**, используй **nav-слой** — там в pos COMMA и DOT подложены скан-коды `LS(/)` и `/`, которые ЙЦУКЕН транслирует как `,` и `.`. Знак вопроса на обеих раскладках набирается обычным способом через Shift (`Shift+/` на EN через base, `Shift+7` на RU через sym-слой).

### Mac-вид модификаторов

На Mac в System Settings → Keyboard → Keyboard Shortcuts → Modifier Keys включён swap **Ctrl ↔ Cmd** (per-keyboard, только для этой клавиатуры). Поэтому firmware-LCTRL превращается в ⌘ Cmd, а firmware-LGUI — в ⌃ Ctrl. Итоговое расположение модификаторов:

```
╭──────────┬─────┬─────┬─────┬─────┬─────╮   ╭─────┬─────┬─────┬─────┬─────┬──────────╮
│  TAB/⌃   │  Q  │  W  │  E  │  R  │  T  │   │  Y  │  U  │  I  │  O  │  P  │    [     │
├──────────┼─────┼─────┼─────┼─────┼─────┤   ├─────┼─────┼─────┼─────┼─────┼──────────┤
│  ESC/⌘   │  A  │  S  │  D  │  F  │  G  │   │  H  │  J  │  K  │  L  │  ;  │    '     │
├──────────┼─────┼─────┼─────┼─────┼─────┤   ├─────┼─────┼─────┼─────┼─────┼──────────┤
│    ⇧     │  Z  │  X  │  C  │  V  │  B  │   │  N  │  M  │  ,  │  .  │  /  │    ⌥     │
╰──────────┴─────┴─┬───┴─┬───┴───┬─┴─────╯   ╰─────┴┬────┴┬────┴──┬──┴───┬─┴──────────╯
                   │  ⌘  │  SYM  │SPACE│            │ENTER│🌐/NAV │ BSPC │
                   ╰─────┴───────┴─────╯            ╰─────┴───────┴──────╯
```

| Mac модификатор | Позиция на клавиатуре |
|-----------------|----------------------|
| ⌘ Cmd           | Левый thumb (крайний) + hold на ESC home-row edge |
| ⌃ Ctrl          | Hold на TAB в левом outer top |
| ⌥ Option        | Правый нижний угол (pinky) |
| ⇧ Shift         | Левый bottom-row edge |
| Fn              | **нет** — F1-F12 и медиа на слое L3 sys |

**Важные маковские сочетания:**
- **⌘C / ⌘V / ⌘Z** — левый thumb (⌘) + Z/X/C/V указательным/средним/безымянным. Одной левой рукой.
- **⌘Tab** (переключение окон) — левый thumb (⌘) + левый pinky на pos 1 (Tab). Одной левой рукой.
- **⌘Space** (Spotlight) — hold ESC home-row edge (⌘) левым pinky + Space левым thumb. Thumb и pinky одной левой.
- **⌘Q / ⌘W** — левый thumb (⌘) + Q/W указательным.
- **⌥←/⌥→** (jump word) — правый pinky в углу (⌥) + hold nav-thumb + левая рука по HJKL/стрелкам на nav.

> Swap настраивается **per-keyboard** в macOS: System Settings → Keyboard → Keyboard Shortcuts → Modifier Keys → выбрать Jorne в дропдауне → поменять ⌃ и ⌘ местами. Обычная клавиатура (встроенная на MacBook) остаётся без изменений.

**Почему так:**
- **Space слева, Enter справа** — пробел нажимается чаще, левый большой палец сильнее у правшей. Enter логичен справа — завершает ввод.
- **ESC/CTRL на home-row edge** — tap = Escape (vim), hold = Ctrl. Совпадает с CapsLock на обычной клавиатуре через AHK/Karabiner.
- **TAB/LGUI в левом outer top** — привычное место «слева от Q», как на обычной клавиатуре. Тап = Tab, hold = LGUI (Cmd/Win). Cmd+Tab: левый thumb (⌘ на LCTRL thumb) + левый pinky на pos 1 (Tab). Alt+Tab: правый pinky на LALT в углу + левый pinky на pos 1 (Tab).
- **LCTRL на thumb** — Ctrl+Z/X/C/V выполняются одной левой рукой (thumb + указательный/средний).
- **F24 на nav-thumb (тап)** — hold = слой NAV, тап = переключение языка через скрипт (AHK/Karabiner). F24 выбран вместо RALT, потому что Windows в русской раскладке конвертирует RAlt в Ctrl+Alt (AltGr), из-за чего скрипт не ловил нажатие. F24 — «пустая» клавиша, которую ни одна раскладка не перехватывает.
- **BSPC на правом крайнем thumb — чистый `&kp BSPC`** без hold-tap. Backspace очень частое действие и его нужно безопасно зажимать для удаления нескольких символов подряд — любая mod-tap обвязка здесь ломает повтор.
- **LALT в правом нижнем углу** — отдельный pinky-ключ без hold-tap. Alt+Tab: правый pinky (LALT) + левый pinky (Tab на pos 1). Alt+стрелки: hold LALT + hold nav-thumb + правая рука по HOME/PGDN/PGUP/END или левая по HJKL через nav-слой.
- **Х (`[`) вернулась на правый pinky top** — своё «родное» место в ЙЦУКЕН. Без hold-tap.
- **Все 31 русская буква на base слое** (без Ё и Ъ) — печатать на русском можно не заходя на другие слои.

## L1 — Символы + Numpad-калькулятор

Активируется **левым средним thumb** (`&mo 1`). Левая рука зажимает слой — правая свободна для набора.

```
╭───────┬─────┬─────┬─────┬─────┬─────╮   ╭──────┬──────┬──────┬───────┬──────┬──────╮
│   `   │  !  │  @  │  #  │  $  │  %  │   │  /   │  7   │  8   │   9   │  -   │ DEL  │
├───────┼─────┼─────┼─────┼─────┼─────┤   ├──────┼──────┼──────┼───────┼──────┼──────┤
│   '   │  ^  │  &  │  ;  │  (  │  )  │   │  *   │  4   │  5   │   6   │  +   │  =   │
├───────┼─────┼─────┼─────┼─────┼─────┤   ├──────┼──────┼──────┼───────┼──────┼──────┤
│ SHIFT │  \  │  [  │  ]  │  <  │  >  │   │  0   │  1   │  2   │   3   │  .   │  ,   │
╰───────┴─────┴─┬───┴─┬───┴─┬───┴─┬───╯   ╰────┬─┴───┬──┴───┬──┴──┬────┴──────┴──────╯
                │     │▓▓▓▓▓│     │            │     │ SYS  │     │
                ╰─────┴─────┴─────╯            ╰─────┴──────┴─────╯
```

С зажатым Shift на sym-слое получаются дополнительные символы (пустые ячейки — с Shift выдают то же что без него):

```
╭───────┬─────┬─────┬─────┬─────┬─────╮   ╭──────┬──────┬──────┬───────┬──────┬──────╮
│   ~   │     │     │     │     │     │   │  ?   │  &   │  *   │   (   │  _   │      │
├───────┼─────┼─────┼─────┼─────┼─────┤   ├──────┼──────┼──────┼───────┼──────┼──────┤
│   "   │     │     │  :  │     │     │   │      │  $   │  %   │   ^   │      │  +   │
├───────┼─────┼─────┼─────┼─────┼─────┤   ├──────┼──────┼──────┼───────┼──────┼──────┤
│       │  |  │  {  │  }  │     │     │   │  )   │  !   │  @   │   #   │  >   │  <   │
╰───────┴─────┴─┬───┴─┬───┴─┬───┴─┬───╯   ╰────┬─┴───┬──┴───┬──┴──┬────┴──────┴──────╯
                │     │▓▓▓▓▓│     │            │     │ SYS  │     │
                ╰─────┴─────┴─────╯            ╰─────┴──────┴─────╯
```

Так видно что на sym + Shift доступны символы, которых нет на чистом sym: `~` `?` `_` `{` `}` `|` (через shift-пары), а цифры numpad'а превращаются в свои shifted-формы (`!@#$%^&*()`). Уже-шифтованные клавиши (`!@#$%^&*()+|<>`) с Shift остаются собой — показаны пустыми.

**Почему так:**

- **Левая сторона — символы для программирования:**
  - **`` ` `` (GRAVE) в левом outer top** — привычная угловая позиция. Tilde `~` доступен как Shift+`` ` ``.
  - **LGUI переехал на base** (на TAB как hold) — на sym-слое home-row edge занят `'` (см. ниже).
  - **Верхний ряд:** `! @ # $ %` — Shift+цифра символы. Мышечная память с обычной клавиатуры.
  - **Home row: `' ^ & ; ( )`** — «shift-символы» 6..0 (`^ & * ( )`) обрамлены `'` и `;` по краям. `;` и `'` также есть на base на правой руке (pos 24, 25) — на sym они дублируют базовые для удобства набора кода не выходя из слоя. Если понадобится место под новые символы, `;` и `'` можно убрать первыми — они всегда доступны на base. `*` на левой руке убран (остался на правой pos 20 и через Shift+N8), `|` убран (Shift+`\` на row 3).
  - **Shifted-пары:** Shift+`'` = `"`, Shift+`;` = `:` — то есть строки, тип-аннотации и словари полностью набираются на sym без переключения.
  - **Bottom row: `\ [ ] < >`** — квадратные скобки (`{` `}` берутся через Shift+`[` `]`), угловые рядом — для HTML/JSX и дженериков.
  - **DEL в правом pinky top** — forward delete рядом с редактируемым текстом, когда левый thumb держит sym.
- **Правая сторона — numpad-калькулятор:**
  - **Центр (U-I-O / J-K-L / M-,-.):** 7-8-9 / 4-5-6 / 1-2-3 — стандартная раскладка калькулятора.
  - **Левый столбец (Y/H/N):** `/` `*` `0` — деление, умножение, ноль внизу.
  - **Правый столбец (P/;):** `-` `+` `=` — вычитание, сложение, результат.
  - **Крайний правый:** DEL / `=` / `,` — forward delete, результат, разделитель.
  - **Нижний правый угол:** `.` и `,` — десятичная точка и разделитель для калькулятора.

**Замечание про русский:** sym-слой (скобки, операторы, цифры, пунктуация) корректно работает только в **английской раскладке** ОС — в русской коды клавиш транслируются в буквы. Это стандартная практика — код и числа вводятся в английском режиме. Для русской пунктуации (`.`/`,`/`?` на русской раскладке) используется nav-слой.

## L2 — Vim-навигация (одной рукой)

Активируется **hold'ом правого среднего thumb** (`lt 2 F24` — тап переключает язык, удержание включает слой). Навигация выполняется **одной правой рукой** — правый thumb зажимает слой, правые пальцы нажимают стрелки.

```
╭───────┬─────┬─────┬─────┬─────┬─────╮   ╭──────┬──────┬──────┬───────┬──────┬──────╮
│       │     │     │     │     │     │   │ HOME │ PGDN │ PGUP │  END  │      │ DEL  │
├───────┼─────┼─────┼─────┼─────┼─────┤   ├──────┼──────┼──────┼───────┼──────┼──────┤
│       │     │     │     │     │     │   │  ←   │  ↓   │  ↑   │   →   │      │      │
├───────┼─────┼─────┼─────┼─────┼─────┤   ├──────┼──────┼──────┼───────┼──────┼──────┤
│       │     │     │     │     │     │   │      │      │,(RU) │ .(RU) │      │      │
╰───────┴─────┴─┬───┴─┬───┴─┬───┴─┬───╯   ╰────┬─┴───┬──┴──┬───┴─┬─────┴──────┴──────╯
                │     │ SYS │     │            │     │▓▓▓▓▓│     │
                ╰─────┴─────┴─────╯            ╰─────┴─────┴─────╯
```

**Почему так:**

- **Home row HJKL: ← ↓ ↑ →** — классическое Vim-расположение. Навигация без отрыва от home row.
- **Верхний ряд: HOME PGDN PGUP END** — вертикально парные со стрелками (← ↔ HOME, ↓ ↔ PGDN, ↑ ↔ PGUP, → ↔ END) прямо над ними. Компактный 2×4 блок навигации.
- **Bottom row: `,(RU)`, `.(RU)`** — русская пунктуация на тех же физических позициях что на английской базе (`,`, `.`). Внутри скан-коды `LS(/)` и `/` — ЙЦУКЕН ОС транслирует их как `,` и `.`. Когда нужно напечатать точку/запятую на русском: hold nav-thumb + привычная позиция. На английской раскладке эти же клавиши выдадут `?` и `/` (побочный эффект, не мешает). `?` на RU делается обычно через Shift+7 на sym-слое.
- **Одноручная навигация** — `lt 2 F24` на правом thumb, стрелки/HOME/END на правых пальцах. Левая рука полностью свободна (например, для мыши).
- **Левая сторона пустая** — чистый навигационный слой, без случайных нажатий.
- **DEL справа вверху** — быстрое удаление при навигации.

## L3 — Sys (Медиа + Bluetooth + F-keys + Система)

Активируется **зажатием обоих layer-ключей** одновременно (L1 + L2). Защита от случайного нажатия — нужно намеренно зажать оба thumb.

```
╭────────┬──────┬──────┬──────┬────────┬────────╮   ╭────────┬─────┬─────┬─────┬───────┬────────╮
│  MUTE  │ PREV │  PP  │ NEXT │ VOL UP │ BRI UP │   │  F1    │ F2  │ F3  │ F4  │  F5   │  F6    │
├────────┼──────┼──────┼──────┼────────┼────────┤   ├────────┼─────┼─────┼─────┼───────┼────────┤
│        │ BT0  │ BT1  │ BT2  │  BT3   │  BT4   │   │  F7    │ F8  │ F9  │ F10 │  F11  │  F12   │
├────────┼──────┼──────┼──────┼────────┼────────┤   ├────────┼─────┼─────┼─────┼───────┼────────┤
│  BOOT  │RESET │      │      │ VOL DN │ BRI DN │   │ BT CLR │ OUT │PSCRN│     │ RESET │  BOOT  │
╰────────┴──────┴──┬───┴──┬───┴──┬─────┴────────╯   ╰────────┴───┬─┴───┬─┴─────┴───────┴────────╯
                   │      │     │      │               │      │     │UNLOCK│
                   ╰──────┴─────┴──────╯               ╰──────┴─────┴──────╯
```

**Почему так:**

- **Conditional layer** — не тратит отдельную клавишу. Медиа и BT используются редко, двойное зажатие — адекватная цена.
- **Левая сторона — медиа и Bluetooth:**
  - **Медиа** на верхнем ряду — MUTE, prev/play-pause/next на удобных позициях.
  - **BT профили на home row** — частое переключение между устройствами на сильных пальцах.
  - **Громкость и яркость вертикально** — VOL UP/DN и BRI UP/DN как слайдеры вверх/вниз.
- **Правая сторона — F-keys и система:**
  - **F1-F6** на верхнем ряду, **F7-F12** на home row — два ряда по 6.
  - **UNLOCK (studio_unlock)** — на правом thumb. При L3 оба средних thumb заняты, правый крайний свободен.

### Пояснения к клавишам L3

**Bluetooth:**

- **BT0–BT4** — переключение между 5 Bluetooth-профилями. Каждый профиль хранит сопряжение с одним устройством. Например: BT0 = ноутбук, BT1 = десктоп, BT2 = планшет. Нажатие переключает активный профиль — клавиатура начинает работать с соответствующим устройством.
- **BT CLR** — очищает сопряжение **текущего** профиля (не всех). Чтобы переподключиться к устройству: выбрать профиль (BT0–BT4) → нажать BT CLR → на устройстве удалить старое сопряжение → подключиться заново.
- **OUT** (`&out OUT_TOG`) — переключает вывод между **Bluetooth** и **USB**. Если клавиатура подключена по USB и по BT одновременно, эта клавиша выбирает, куда отправлять нажатия. По умолчанию приоритет у USB.

**Система:**

- **BOOT** (`&bootloader`) — перевод контроллера в режим загрузчика (как двойной клик RESET). Появится USB-диск для прошивки `.uf2`. Расположен зеркально в углах обеих сторон — чтобы можно было прошить любую половину.
- **RESET** (`&sys_reset`) — перезагрузка контроллера. Не стирает прошивку, просто рестарт. Тоже зеркально в углах.
- **PSCRN** (`Print Screen`) — скриншот (только Windows/Linux, на macOS не работает — используйте Cmd+Shift+3/4/5).
- **UNLOCK** (`&studio_unlock`) — разблокировка ZMK Studio для редактирования раскладки через браузер. Подробнее в разделе ZMK Studio.

**Медиа:**

- **MUTE** — отключение/включение звука.
- **PREV / PP / NEXT** — предыдущий трек, play/pause, следующий трек.
- **VOL UP / VOL DN** — громкость.
- **BRI UP / BRI DN** — яркость экрана (работает на ноутбуках).
- **BOOT и RESET** — зеркально в углах обеих сторон для защиты от случайного нажатия.

## L4 — Игровой слой

Слой для игр и навигации (проводник, файловые менеджеры). Работает как **toggle** — включается один раз и остаётся активным, пока не выключишь. Все hold-tap поведения отключены — каждая клавиша срабатывает мгновенно, без задержек.

### Как включить / выключить

**Combo:** быстро нажать одновременно **LCTRL + BSPC** (два крайних thumb — левый крайний + правый крайний). Обе клавиши должны быть нажаты в пределах 50мс. Тот же combo выключает игровой слой обратно.

Также можно выключить нажатием **правого среднего thumb** (EXIT на game слое).

### Раскладка

```
╭───────┬─────┬─────┬─────┬─────┬─────╮   ╭──────┬──────┬──────┬───────┬──────┬──────╮
│  ESC  │  Q  │  W  │  E  │  R  │  T  │   │  1   │  2   │  ↑   │   3   │  4   │ BSPC │
├───────┼─────┼─────┼─────┼─────┼─────┤   ├──────┼──────┼──────┼───────┼──────┼──────┤
│  TAB  │  A  │  S  │  D  │  F  │  G  │   │ HOME │  ←   │  ↓   │   →   │ END  │  F2  │
├───────┼─────┼─────┼─────┼─────┼─────┤   ├──────┼──────┼──────┼───────┼──────┼──────┤
│ LSHFT │  Z  │  X  │  C  │  V  │  B  │   │  5   │  6   │  7   │   8   │  9   │  0   │
╰───────┴─────┴─┬───┴─┬───┴─┬───┴─────╯   ╰──────┴───┬──┴───┬──┴───────┴──────┴──────╯
                │LCTRL│LALT │SPACE│           │ENTER│ EXIT │ DEL  │
                ╰─────┴─────┴─────╯           ╰─────┴──────┴──────╯
```

### Почему так

- **Левая сторона — WASD-зона:**
  - Полностью повторяет стандартную клавиатуру: QWERT / ASDFG / ZXCVB на тех же физических позициях.
  - **Без hold-tap** — TAB это просто TAB (не TAB/GUI), LSHFT это просто LSHFT. Мгновенный отклик.
  - **LCTRL и LALT на thumb** — основные модификаторы для игр (приседание, альтернативное действие).
  - **SPACE** — прыжок.
- **Правая сторона — навигация + цифры:**
  - **Стрелки крестом** — ↑ над ↓, ← и → по бокам. Для игр со стрелочным управлением и навигации в проводнике.
  - **HOME / END** — обрамляют стрелки слева и справа. Переход к началу/концу списка в проводнике.
  - **F2** рядом с END — переименование файлов в проводнике.
  - **BSPC** справа вверху — удаление при вводе.
  - **Цифры 1-4 вверху, 5-0 внизу** — быстрые слоты в играх, нумерация.
- **Правый thumb:**
  - **ENTER** — как на base слое, подтверждение действий.
  - **EXIT** — выход из game слоя (та же позиция, что mo(2) — легко найти на ощупь).
  - **DEL** — удаление файлов в проводнике.

## Настройка ОС (Windows + macOS)

Сплит-клавиатура и обычная клавиатура используют **одни и те же скрипты** на стороне ОС. Скрипты не конфликтуют между собой — каждая клавиатура отправляет разные keycodes.

### Концепция: CapsLock как Escape/Ctrl для Vim

В Vim клавиша **Escape** используется постоянно (выход из Insert mode), а **Ctrl** нужен для множества команд (`C-[` = Escape, `C-r`, `C-w`, `C-a` и т.д.). На стандартной клавиатуре обе эти клавиши расположены неудобно — Escape далеко вверху, Ctrl в углу.

Решение (де-факто стандарт в Vim-сообществе): **CapsLock** на позиции home row превращается в двойную клавишу:
- **Тап** (короткое нажатие) → **Escape** — мгновенный выход из Insert mode, не зависит от языковой раскладки
- **Удержание** → **Ctrl** — все Ctrl-комбинации мизинцем с home row, без выворачивания руки к углу клавиатуры

Дополнительно, **переключение языка** переносится на клавишу под правым большим пальцем (RAlt/Right Cmd) — естественное движение, не мешает набору текста.

На сплит-клавиатуре это реализовано в **прошивке**: home-row edge = ESC (tap) / CTRL (hold). На обычной клавиатуре — через скрипты (AHK/Karabiner). Руки делают одно и то же на обеих клавиатурах.

| Действие | Обычная клавиатура | Сплит-клавиатура |
|----------|-------------------|-----------------|
| Escape (vim) | CapsLock tap → скрипт → Esc | ESC/CTRL tap → ESC (прошивка) |
| Ctrl (home row) | CapsLock hold → скрипт → Ctrl | ESC/CTRL hold → Ctrl (прошивка) |
| Переключение языка | RAlt tap → скрипт → смена языка | F24 tap → скрипт → смена языка |
| Alt модификатор | RAlt hold → скрипт → LAlt | LALT hold (прошивка) |

**Почему нет конфликта:** сплит отправляет **ESC** напрямую (не CapsLock) — скрипт CapsLock его не перехватывает. Сплит отправляет **F24** для переключения языка (не RAlt) — скрипт RAlt его не перехватывает. Обычная клавиатура отправляет **RAlt** — скрипт F24 его не перехватывает. Два разных пути, без пересечений.

> **Почему F24 вместо RAlt:** Windows в русской раскладке конвертирует RAlt в Ctrl+Alt (AltGr-поведение), из-за чего AHK-скрипт не ловил нажатие. F24 — «пустая» клавиша без побочных эффектов в любой раскладке.

### Windows — AutoHotkey

Установить [AutoHotkey v2](https://www.autohotkey.com/). Создать файл `keyboard.ahk` и добавить в автозагрузку:

```autohotkey
#Requires AutoHotkey v2.0

; CapsLock: tap → Escape, hold → Ctrl
*CapsLock:: {
    SendInput("{LCtrl down}")
    KeyWait("CapsLock")
    SendInput("{LCtrl up}")
    if (A_PriorKey = "CapsLock") {
        SendInput("{Escape}")
    }
}

; Правый Alt (обычная клавиатура): tap → переключение языка, hold → Left Alt
*RAlt:: {
    if KeyWait("RAlt", "T0.2") {
        ; Отпущена быстро (< 200мс) — tap → переключить язык
        if (A_PriorKey = "RAlt")
            SwitchLanguage()
    } else {
        ; Удерживается (> 200мс) — hold → отправить LAlt
        SendInput("{LAlt down}")
        KeyWait("RAlt")
        SendInput("{LAlt up}")
    }
}

; F24 (сплит-клавиатура): переключение языка
F24::SwitchLanguage()

; Переключение языка через Windows API (без симуляции клавиш)
SwitchLanguage() {
    hwnd := WinExist("A")
    threadId := DllCall("GetWindowThreadProcessId", "Ptr", hwnd, "Ptr", 0, "UInt")
    hkl := DllCall("GetKeyboardLayout", "UInt", threadId, "Ptr")
    langId := hkl & 0xFFFF
    if (langId = 0x0409)
        PostMessage(0x50, 0, DllCall("LoadKeyboardLayout", "Str", "00000419", "UInt", 1, "Ptr"),, "ahk_id " hwnd)
    else
        PostMessage(0x50, 0, DllCall("LoadKeyboardLayout", "Str", "00000409", "UInt", 1, "Ptr"),, "ahk_id " hwnd)
}

```

### macOS — Karabiner-Elements

Установить [Karabiner-Elements](https://karabiner-elements.pqrs.org/). Добавить два правила в Complex Modifications:

**CapsLock → Escape (tap) / Ctrl (hold):**

```json
{
    "description": "CapsLock → Escape (tap) / Ctrl (hold)",
    "manipulators": [
        {
            "from": {
                "key_code": "caps_lock",
                "modifiers": { "optional": ["any"] }
            },
            "to": [{ "key_code": "left_control" }],
            "to_if_alone": [{ "key_code": "escape" }],
            "type": "basic"
        }
    ]
}
```

**Right Cmd (обычная клавиатура) → переключение языка (tap) / Cmd (hold):**

```json
{
    "description": "Right Cmd → switch language (tap) / Cmd (hold)",
    "manipulators": [
        {
            "conditions": [
                {
                    "input_sources": [{ "language": "en" }],
                    "type": "input_source_if"
                }
            ],
            "from": {
                "key_code": "right_command",
                "modifiers": { "optional": ["any"] }
            },
            "to": [{ "key_code": "right_command", "lazy": true }],
            "to_if_alone": [{ "select_input_source": { "language": "ru" } }],
            "type": "basic"
        },
        {
            "conditions": [
                {
                    "input_sources": [{ "language": "ru" }],
                    "type": "input_source_if"
                }
            ],
            "from": {
                "key_code": "right_command",
                "modifiers": { "optional": ["any"] }
            },
            "to": [{ "key_code": "right_command", "lazy": true }],
            "to_if_alone": [{ "select_input_source": { "language": "en" } }],
            "type": "basic"
        }
    ]
}
```

> Karabiner переключает язык через macOS API (`select_input_source`), а не через клавиатурные шорткаты — это надёжнее. `"lazy": true` предотвращает отправку Cmd при одиночном tap.

**F24 (сплит-клавиатура) → переключение языка:**

```json
{
    "description": "F24 → switch language (split keyboard)",
    "manipulators": [
        {
            "conditions": [
                {
                    "input_sources": [{ "language": "en" }],
                    "type": "input_source_if"
                }
            ],
            "from": { "key_code": "f24" },
            "to": [{ "select_input_source": { "language": "ru" } }],
            "type": "basic"
        },
        {
            "conditions": [
                {
                    "input_sources": [{ "language": "ru" }],
                    "type": "input_source_if"
                }
            ],
            "from": { "key_code": "f24" },
            "to": [{ "select_input_source": { "language": "en" } }],
            "type": "basic"
        }
    ]
}
```

### Особенности macOS

**Модификаторы (Ctrl ↔ Cmd):** на macOS основной модификатор — Cmd (⌘ = LGUI). LGUI на hold-tap TAB/LGUI, LCTRL на thumb и ESC/CTRL. При необходимости можно поменять Control и Command для сплит-клавиатуры: **System Settings → Keyboard → Keyboard Shortcuts → Modifier Keys**. Это per-keyboard настройка.

**HOME / END:** на macOS = начало/конец документа, не строки. Для строки: Cmd+←/→. Или добавить правило "PC-style Home/End" в Karabiner.

**Print Screen (PSCRN):** не работает на macOS. Скриншоты: Cmd+Shift+3 (экран), Cmd+Shift+4 (область), Cmd+Shift+5 (панель).

**Медиа-клавиши:** громкость, яркость, prev/play/next — работают без настройки.

## Решение проблем

| Проблема | Причина | Решение |
|----------|---------|---------|
| Клавиатура не видна по BT | Прошита `settings_reset` | Прошить основную прошивку на обе половины |
| Клавиатура не видна по BT | Старое BT-сопряжение на Mac | Удалить устройство и подключиться заново |
| Клавиатура не видна по BT | `CONFIG_ZMK_POINTING=y` без hardware | Убрать эту опцию из jorne.conf |
| Половинки не связываются | Сбой BLE-бондов | settings_reset на обе, затем основная прошивка |
| Keymap не компилируется | Неправильное количество позиций | 44 позиции (не 42), `&none` по краям первого ряда |
| "No board named 'nice_nano_v2'" | ZMK main переименовал board | Использовать ZMK v0.3.0 |
